Git

Git es un sistema de control de versiones distribuido utilizado principalmente para el seguimiento de cambios en el código fuente durante el desarrollo de software. Fue creado por Linus Torvalds en 2005 y desde entonces ha ganado una gran popularidad entre desarrolladores y equipos de programación en todo el mundo. Con Git, los desarrolladores pueden colaborar en proyectos de software de manera eficiente, manteniendo un historial completo de todos los cambios realizados en el código. Esto permite a los equipos trabajar de manera colaborativa, rastrear quién hizo qué cambios, revertir a versiones anteriores del código si es necesario y fusionar fácilmente las contribuciones de varios desarrolladores en una sola línea de desarrollo. Algunos de los conceptos clave en Git incluyen repositorios (donde se almacena el código), confirmaciones (registros de cambios), ramas (versiones paralelas del código), fusiones (integración de cambios entre ramas) y conflictos (cuando dos cambios entran en conflicto y deben ser resueltos manualmente). Git proporciona una serie de comandos y herramientas para trabajar con estos conceptos de manera efectiva, lo que lo convierte en una herramienta esencial para el desarrollo de software colaborativo.

Algunas de sus etiquetas mas usadas son:

Git init:

Inicializa un repositorio git crea el directorio .git inicial en un proyecto nuevo o existente.

Git clone

Crea una copia de repositorio GIT de una fuente externa. También añade la ubicación original como remota de modo que puedas traerlo de nuevo y lanzarlo si tienes permisos.

Git config

Establece valores de configuración para tu usuario, email, gpg key, algoritmo diff preferido, formatos de archivo y más.

Git add

Añade cambios de archivos en tu directorio de ensayo a tu index.

git commit

Toma todos los cambios escritos en el index, crea un nuevo objeto de confirmación que apunta a él y establece la rama para que apunte a esa nueva confirmación.

Git status

Te muestra el estado de los archivos en el index en comparación con los del directorio de trabajo. Enumerará los archivos que no están rastreados (solo en su directorio de trabajo), modificados (rastreados pero aún no actualizados en tu index), y almacenados (añadidos a tu index y listos para comprometerse).

Git branch

Para listar las ramas existentes, incluyendo las ramas remotas, si se proporciona ‘-a’. Crea una nueva rama si se proporciona un nombre.

Git push

Envía todos los objetos modificados localmente al repositorio remoto.

Git log

Muestra una lista de confirmaciones en una rama que incluye los detalles correspondientes.

HTML5

HTML (HyperText Markup Language) es el lenguaje de marcado estándar utilizado para crear páginas web. Es la base de cualquier página web y se utiliza para estructurar el contenido y definir su significado semántico. HTML utiliza etiquetas para marcar diferentes elementos dentro de una página web, como encabezados, párrafos, imágenes, enlaces, formularios, entre otros.

Etiquetas semanticas

Las etiquetas semánticas en HTML5 son elementos específicos diseñados para proporcionar un significado semántico claro al contenido de una página web. Estas etiquetas ayudan a los motores de búsqueda, navegadores, desarrolladores web y otros usuarios a comprender mejor la estructura y el propósito del contenido

header

Define una sección de encabezado para el contenido, que puede contener logotipos, encabezados, menús de navegación, etc.

nav

Define una sección de navegación, que generalmente contiene enlaces a otras páginas o secciones dentro del sitio web.

main

Define el contenido principal de la página. Debería contener el contenido central y relevante para la página en sí.

article

Define un artículo independiente y autónomo dentro de una página. Por ejemplo, un artículo de blog, una publicación de redes sociales, un comentario, etc.

section

Define una sección temática o agrupación de contenido relacionado dentro de una página. Es una etiqueta genérica y se utiliza cuando no hay una etiqueta semántica más específica disponible.

aside

Define contenido secundario o relacionado que está separado del contenido principal, como barras laterales, anuncios, notas, etc.

footer

Define una sección de pie de página para el contenido, que generalmente contiene información de contacto, enlaces útiles, derechos de autor, etc.

Etiquetas de texto

En HTML, las etiquetas de texto se utilizan para definir y estructurar el contenido textual dentro de un documento web. Estas etiquetas permiten aplicar diferentes estilos, formatos y estructuras al texto para mejorar la legibilidad y la presentación de la página. Aquí hay algunas etiquetas de texto comunes en HTML:

  • p: Define un párrafo de texto.
  • h1 a h6: Define encabezados de diferentes.
  • strong: Aplica estilo de texto en negrita.
  • em: Aplica estilo de texto en cursiva (énfasis).
  • u: Subraya el texto.

Etiquetas de agrupación

Las etiquetas de agrupación en HTML se utilizan para organizar y estructurar el contenido de una página web en diferentes bloques o secciones. Estas etiquetas permiten crear una jerarquía y división lógica del contenido, lo que facilita la comprensión y el estilo de la página. Aquí tienes algunas de las etiquetas de agrupación más comunes en HTML:

  • div: Define una división genérica o contenedor de bloques en un documento HTML.
  • span: Define un contenedor en línea para el texto u otros elementos en línea.
  • header: Define una sección de encabezado para el contenido de una página o una sección.
  • footer: Define una sección de pie de página para el contenido de una página o una sección.
  • nav: Define una sección de navegación para enlaces de navegación principales.
  • main Define el contenido principal o principal de una página.
  • section: Define una sección temática en un documento HTML.
  • article: Define un contenido independiente y autónomo que puede ser distribuido e reutilizado por sí mismo.
  • aside:Define contenido que es tangencial o relacionado con el contenido circundante, como barras laterales.

Etiquetas de multimedia

Las etiquetas de multimedia en HTML se utilizan para incrustar contenido multimedia, como imágenes, audio, video y otros elementos relacionados, en una página web. Estas etiquetas permiten enriquecer el contenido de la página con diferentes tipos de medios. Aquí tienes algunas de las etiquetas de multimedia más comunes en HTML:

  • img: Se utiliza para incrustar imágenes en una página web.
  • audio: Se utiliza para incrustar archivos de audio en una página web.
  • video: Se utiliza para incrustar archivos de video en una página web.
  • source: Se utiliza dentro de las etiquetas audio y video para especificar diferentes fuentes de medios, como formatos de audio o video alternativos.
  • track: Se utiliza dentro de las etiquetas video y audio para especificar pistas de texto como subtítulos, subtítulos en varios idiomas o descripciones de audio.
  • embet: Se utiliza para incrustar contenido externo, como archivos multimedia, dentro de un documento HTML. Es compatible con una amplia variedad de tipos de contenido, como Flash, PDF, etc., aunque su uso se ha vuelto menos común con el tiempo.
  • objet: Se utiliza para incrustar objetos externos, como archivos multimedia, en una página web. Se puede utilizar como alternativa a , pero es más complejo de usar y se prefiere cada vez menos en favor de otras etiquetas como video y audio.

Etiquetas de tablas

Las etiquetas de tablas en HTML se utilizan para crear y mostrar tablas de datos en una página web. Permiten organizar la información en filas y columnas para una mejor visualización y comprensión. Aquí tienes algunas de las etiquetas de tablas más comunes en HTML:

  • table: Define una tabla en HTML.
  • tr: Define una fila en una tabla.
  • th: Define una celda de encabezado de tabla. Se utiliza para los encabezados de columna o fila.
  • td: Define una celda de datos en una tabla. Se utiliza para el contenido de la tabla.
  • caption: Define el título o la leyenda de una tabla.
  • thead: Define el encabezado de una tabla.
  • tbody: Define el cuerpo principal de una tabla.
  • tfoot: Define el pie de una tabla.
  • colgroup: Define un grupo de columnas en una tabla.
  • col: Define las propiedades de una o más columnas en una tabla.

Etiquetas de formularios

Las etiquetas de formularios en HTML se utilizan para crear formularios interactivos en una página web. Estos formularios permiten a los usuarios ingresar y enviar datos al servidor para su procesamiento. Aquí tienes algunas de las etiquetas de formularios más comunes en HTML:

  • form: Define un formulario en HTML.
  • input: Define un campo de entrada en el formulario. Puede tomar diferentes tipos, como texto, contraseña, botones de radio, casillas de verificación, etc.
  • textarea: Define un área de texto de varias líneas para que los usuarios ingresen texto largo.
  • select: Define un menú desplegable o una lista de opciones.
  • option: Define una opción dentro de un elemento select.
  • button: Define un botón que el usuario puede hacer clic para enviar el formulario o realizar alguna otra acción.
  • labelDefine una etiqueta descriptiva para un elemento de formulario. Ayuda a los usuarios a comprender qué se espera que ingresen en un campo de entrada.
  • fieldsetDefine un grupo de controles de formulario y una leyenda opcional.
  • legend: Define una leyenda para el elemento fieldset.

Hojas de estilo en cascada CSS 3

CSS (Cascading Style Sheets) es un lenguaje de estilo utilizado para describir el aspecto y el formato de un documento HTML o XML. Permite definir el diseño, la presentación y la apariencia visual de una página web, separando así la estructura y el contenido del diseño y el estilo. CSS se utiliza para aplicar estilos como colores, fuentes, márgenes, bordes, tamaños, posicionamiento y más a los elementos de una página web.

Tipos de selectores

Los selectores en CSS son patrones que se utilizan para seleccionar y aplicar estilos a elementos HTML específicos dentro de una página web. Los selectores permiten identificar qué elementos deben ser afectados por las reglas de estilo definidas en una hoja de estilos CSS. Hay varios tipos de selectores en CSS, algunos de los más comunes incluyen:

  • Selector de tipos: Selecciona elementos HTML según su tipo. Por ejemplo, p selecciona todos los párrafos en el documento.
  • Selector de clase: Selecciona elementos HTML que tienen un atributo class específico. Se denotan con un punto (.) seguido del nombre de la clase.
  • Selector de ID: Selecciona un único elemento HTML con un atributo id específico. Se denotan con un signo de numeral (#) seguido del nombre del ID.
  • Selector universal: Selecciona todos los elementos HTML en el documento. Se denota por un asterisco (*).
  • Selector de decendenciaSelecciona elementos que son descendientes de otro elemento. Se utiliza un espacio para separar los selectores.
  • Selector de hijo directo: Selecciona elementos que son hijos directos de otro elemento. Se utiliza el signo mayor que (>) para separar los selectores.
  • Selector de atributo: Selecciona elementos que tienen un atributo HTML específico. Se utiliza la siguiente sintaxis: [atributo="valor"].

Pseudo clases

Las pseudoclases en CSS son palabras clave que se añaden a los selectores para especificar un estado especial del elemento seleccionado. Estos estados pueden ser interacciones del usuario, como cuando el usuario pasa el ratón por encima de un elemento, hace clic en él, o incluso cuando un elemento tiene un estado específico como estar activo o seleccionado. Las pseudoclases permiten aplicar estilos a estos estados específicos.

  • hover: Selecciona un elemento cuando el usuario pasa el ratón por encima de él.
  • active: Selecciona un elemento mientras está siendo activado por el usuario, como
  • focus: Selecciona un elemento mientras tiene el foco de entrada, como cuando se selecciona mediante el teclado o se hace clic en él para escribir.
  • visited: Selecciona un enlace que ha sido visitado por el usuario.
  • first-child: Selecciona el primer hijo de un elemento.
  • last-child: Selecciona el último hijo de un elemento.
  • nth-child(n): Selecciona el enésimo hijo de un elemento, donde "n" es un número o una fórmula matemática.
  • nth-of-type(n): Selecciona el enésimo hijo de un tipo específico de elemento.
  • nth-last-child(n): Selecciona el enésimo hijo, comenzando desde el último hijo.
  • nth-last-of-type(n): Selecciona el enésimo hijo de un tipo específico de elemento, comenzando desde el último hijo.
  • not(selector): Selecciona elementos que no coinciden con el selector especificado.
  • checked: Selecciona elementos de formulario, como casillas de verificación o botones de radio, que están marcados o seleccionados.
  • disabled: Selecciona elementos de formulario que están deshabilitados.
  • enabled: Selecciona elementos de formulario que están habilitados.
  • empty: Selecciona elementos que no tienen hijos.
  • required: Selecciona elementos de formulario que tienen el atributo "required" especificado.

Unidades de medida relativas

Las unidades de medida relativas en CSS son unidades que se calculan en relación con otros valores, como el tamaño de la fuente del elemento padre, el ancho del contenedor o la altura de la ventana del navegador. Estas unidades permiten diseñar diseños web de manera más flexible y adaptable, ya que se ajustan automáticamente en función del contexto en el que se utilizan.

  • em: Esta unidad se basa en el tamaño de la fuente del elemento actual. Por ejemplo, si el tamaño de fuente del elemento actual es 16px, 1em sería equivalente a 16px. Si se aplica a un elemento secundario, su tamaño se calculará en relación con el tamaño de fuente de su elemento padre.
  • rem: Similar a em, pero se basa en el tamaño de la fuente del elemento raíz (normalmente el elemento ). Esto lo hace más predecible y fácil de manejar en comparación con em, ya que no depende de la cadena de elementos contenedores.
  • %: Esta unidad se basa en el tamaño del elemento padre. Por ejemplo, si se establece el ancho de un elemento hijo en un 50%, su ancho será la mitad del ancho de su elemento padre.
  • vw (viewport width): Esta unidad se basa en el ancho de la ventana del navegador. 1vw es igual al 1% del ancho de la ventana del navegador.
  • vh (viewport height): Similar a vw, pero se basa en la altura de la ventana del navegador.

Fondos y bordes

En CSS3, los fondos y los bordes son propiedades que permiten agregar estilos visuales a los elementos HTML, como divs, párrafos, encabezados y más. Estas propiedades son fundamentales para diseñar y dar estilo a las interfaces de usuario web. Aquí te explico más sobre ellos:

  • border: Define un borde alrededor del elemento. Puede especificar ancho, estilo y color del borde en una sola propiedad.
  • border-width: Define el ancho del borde.
  • border-style: Define el estilo del borde (por ejemplo, sólido, punteado, etc.).
  • border-color: Define el color del borde.
  • border-radius: Define el radio de la esquina del borde, creando esquinas redondeadas.

Modelo de cajas

El "Modelo de Cajas" en CSS3 es un concepto fundamental que describe cómo se representan y se organiza el contenido en una página web. Básicamente, en CSS, cada elemento HTML se representa como una "caja" rectangular que consiste en contenido, relleno (padding), borde (border) y margen (margin). Este modelo permite controlar la presentación y el diseño de los elementos en una página web.

  • width, height: Controlan el ancho y la altura del contenido de la caja.
  • padding: Define el espacio entre el contenido y el borde.
  • border: Establece el estilo, ancho y color del borde.
  • margin: Define el espacio entre el borde del elemento y otros elementos adyacentes.

Position

En CSS3, la propiedad position se utiliza para controlar la posición de un elemento en una página web. Esta propiedad puede tomar varios valores, cada uno de los cuales determina cómo se posiciona el elemento en relación con su contenedor o con otros elementos en la página.

  • static: Este es el valor predeterminado. Los elementos con posición estática se muestran en el orden normal del flujo del documento. Ignora las propiedades top, right, bottom y left.
  • relative: Los elementos con posición relativa se posicionan en relación con su posición normal. Esto significa que puedes desplazar el elemento utilizando las propiedades top, right, bottom y left, sin afectar la posición de otros elementos en la página. Sin embargo, el espacio que ocupa el elemento permanece reservado en el flujo del documento.
  • absolute: Los elementos con posición absoluta se posicionan en relación con el elemento padre más cercano que tenga una posición diferente a static. Si no hay ningún elemento padre con una posición diferente a static, el elemento se posicionará en relación con el viewport (la ventana del navegador). Los elementos con posición absoluta no afectan el diseño de otros elementos y se pueden superponer.
  • fixed: Similar a absolute, pero los elementos con posición fija se posicionan en relación con el viewport en lugar de un elemento padre. Esto significa que el elemento permanecerá fijo en su posición incluso cuando la página se desplace.
  • sticky: Los elementos con posición pegajosa se comportan como relativos hasta que se desplaza más allá de cierto umbral definido por las propiedades top, right, bottom o left, momento en el cual se comportarán como fixed y permanecerán fijos en esa posición mientras la página se desplaza.

Display

En CSS3, la propiedad display determina cómo se muestra un elemento HTML en la página web. Es una propiedad fundamental que controla el comportamiento de diseño de un elemento y cómo interactúa con otros elementos en el documento.

  • block: Los elementos con display: block se comportan como bloques en línea que ocupan todo el ancho disponible en su contenedor y comienzan en una nueva línea. Ejemplos de etiquetas HTML que pueden tener display: block incluyen div, p, h1 a h6, ul, ol, etc.
  • inline: Los elementos con display: inline se muestran en la misma línea que el contenido circundante y solo ocupan el espacio necesario para su contenido. Ejemplos de etiquetas HTML que pueden tener display: inline incluyen span, a, strong, em, img, etc.
  • inline-block: Combina las características de elementos en línea y en bloque. Los elementos con display: inline-block se comportan como elementos en línea, pero permiten establecer propiedades de ancho y altura. Esto los hace útiles para crear diseños más complejos sin perder la capacidad de alinear elementos horizontalmente. Por ejemplo, los elementos input y button suelen tener display: inline-block.
  • flex: Introduce un modelo de diseño flexible que facilita el posicionamiento y el ajuste del tamaño de los elementos dentro de su contenedor. Los elementos con display: flex se distribuyen de manera flexible dentro de su contenedor, lo que permite controlar el tamaño, el orden y la alineación de los elementos hijos. Esto es especialmente útil para diseñar diseños complejos y responsivos. Puedes aplicar esta propiedad a cualquier elemento HTML, como div, section, etc.
  • grid: Introduce un modelo de diseño basado en rejillas que permite organizar los elementos en filas y columnas. Los elementos con display: grid se colocan en una cuadrícula definida por el desarrollador, lo que facilita la creación de diseños más complejos y estructurados. Esto es especialmente útil para diseños más elaborados que requieren un posicionamiento preciso de los elementos. Puedes aplicar esta propiedad a cualquier elemento HTML, como div, section, etc.

Responsive Design “Media Queries”

Los "Media Queries" son una característica importante en CSS3 que permiten crear diseños web receptivos o responsive design. Con media queries, puedes aplicar estilos específicos basados en las características del dispositivo o las características de visualización del navegador, como el tamaño de la pantalla, la orientación, la densidad de píxeles, etc. Esto te permite crear diseños web que se adapten y respondan de manera inteligente a diferentes dispositivos y tamaños de pantalla.

JavaScript

JavaScript es un lenguaje de programación de alto nivel que se utiliza principalmente en el desarrollo web para agregar interactividad y dinamismo a las páginas web. Es un lenguaje interpretado, lo que significa que el código fuente se ejecuta directamente por un intérprete o navegador web sin necesidad de ser compilado previamente.
